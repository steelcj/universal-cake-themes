# UniversalCake Theme Values and Human Evaluation Framework

This repository contains theme explorations, prototypes, and reference implementations developed within the UniversalCake ecosystem.

Before describing how themes are evaluated, we begin with the values that shape how they are built.

While system themes are considered to be aesthetic exercises, they are also expressions of ethical and architectural commitments to accessibility and functionality. As an organization dedicated to well being we want to ensure that we strive to serve the real population of real people in order to promote fairer access to our cultural, economic, political and social spaces.

## Core Values

### Accessibility as Ethical Responsibility

Accessibility is frequently treated as an enhancement layer, but it would be better served as a baseline obligation to those who want to strive to serve our highly diverse population.

We strive to develop themes where:

- Content remains readable under high zoom.
- Keyboard navigation works.
- Focus states remain visible.
- Contrast options meets or exceed WCAG AA standards.
- Motion that respects user preferences.

Accessibility is treated as care for human systems, not merely compliance.

### Agnosticism as Resilience

We strive to create themes that function across environments:

- Multiple browser engines
- Mobile and desktop viewports
- With or without JavaScript
- Without reliance on fragile external services or subscriptions.

Agnostic design protects us against:

- Platform lock-in
- Browser-specific rendering failures
- Dependency decay over time

Ideally our themes degrade gracefully and age well.

### Cognitive Clarity as Design Discipline

UniversalCake prioritizes readable, humane design that supports multiple human languages.

Themes aim to:

- Reduce visual noise
- Preserve structural hierarchy
- Support long-form reading
- Avoid spectacle-driven layout

Design restraint is intentional.

Clarity is considered a form of respect.

### Longevity over Trend

Themes are not optimized for fashion cycles.

They are developed with the expectation that:

- They should still feel appropriate in five to ten years.
- They should not rely on visual gimmicks.
- They should prioritize stability over novelty.

## From Values to Development Practice

These values influence development decisions:

- Minimal JavaScript for core functionality
- Semantic HTML as foundation
- Progressive enhancement
- Standards-based CSS
- Clear typographic hierarchy
- Predictable spacing systems
- Limited color palettes

Accessibility and resilience take precedence.

## Human Evaluation Framework

Although automated testing can be helpful in ensuring for quality, human evaluation is vital to ensure for real world practicalities and elegant designs that better serve humans and, in some cases, automated systems that provide for valuable and desired services such as documentation and learning systems.

Themes can be reviewed and assigned qualitative ratings across the following three dimensions:

- Accessibility
- Agnosticism
- Cognitive Clarity

Ratings use descriptive categories rather than numeric scores:

- Excellent
- Strong
- Adequate
- Needs Work

## Evaluation Dimensions

### Accessibility

Human reviewers assess:

- Logical heading structure
- Proper landmark usage
- Keyboard navigability
- Visibility of focus states
- Contrast safety
- Respect for reduced-motion preferences

An **Excellent** rating indicates strong compliance with accessibility principles and calm interaction design.

### Agnosticism

Reviewers can consider:

- Cross-browser consistency
- Mobile integrity at narrow widths
- Functional rendering without JavaScript
- Minimal dependency chains
- Standards-based markup

An **Excellent** rating indicates strong resilience across environments.

### Cognitive Clarity

Reviewers evaluate:

- Visual hierarchy
- Predictable spacing
- Readability of long-form content
- Color restraint
- Absence of unnecessary visual noise

An **Excellent** rating indicates sustained reading comfort and low cognitive friction.

## Certification Tiers

Themes may be grouped into the following tiers:

### UC-Core

Accessible, resilient, calm, and production-ready.

### UC-Stable

Strong alignment with values, with minor tradeoffs or opinionated choices.

### UC-Experimental

Exploratory or stylistically bold. Preserved for learning and experimentation

### UC-Archive

Retained for reference. Likely require integration for specific content generators. Not recommended for deployment as is.

---

## On Measurement

We recognize that:

- Accessibility can be partially automated.
- Cross-browser stability can be tested.
- Performance can be measured.

We keep in mind that things like:

- Calmness cannot be reduced to a number.
- Ethical tone is difficult and in some cases cannot be quantified.
- Human experience remains central.

We use automation to supports our values. If may support, but should never replace human judgment.

## Closing Principle

UniversalCake themes are developed under the belief that:

* Careful design and structure takes longer but can significantly reduces friction over time.
* Resilient design protects users.
* Presentation can and should supports wellbeing.  
* Evaluation is not about scoring superiority. It is about maintaining alignment with these commitments.
